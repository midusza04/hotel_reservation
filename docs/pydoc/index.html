Modul

actors

Ray actors implementing the distributed hotel reservation domain.

The actors split responsibility between hotel inventory, booking orchestration,
payments, user history, metrics, audit logging and administration. Each actor
owns its own state, which matches Ray's actor model and avoids shared memory
between distributed components.

db

Database models and persistence helpers for the reservation system.

The actors keep operational state in memory, while this module provides durable
snapshots and append-only audit data in Postgres through SQLAlchemy.

Modul

main

FastAPI entrypoint for the Ray-based hotel reservation system.

The module owns HTTP routing, demo authentication, application lifecycle
initialization and the bridge between synchronous API handlers and Ray actors.
Ray actors keep domain state and execute distributed reservation workflows,
while FastAPI exposes a small JSON API consumed by the demo frontend.
