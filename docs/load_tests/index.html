1. Wprowadzenie

Testy obciazeniowe symuluja wielu rownoczesnych uzytkownikow HTTP i mierza latencje, throughput oraz poprawnosc pod obciazeniem. Locust komunikuje sie wylacznie z API (http://localhost:8000) — nie wymaga lokalnej instalacji Ray.

pip install locust
docker compose up -d
python tests/load/seed_hotels.py --rooms 500
locust -f tests/load/locustfile.py --host http://localhost:8000 --headless -u 30 -r 5 -t 60s --csv tests/load/results/run1

Konta testowe: user1–user50 / pass, admin / admin.

5. Contention test

Pure Wielu userow rezerwuje ten sam pokoj jednoczesnie. Bez cancel, bez search. Cel: h-waw-1/single, seed --rooms 1.

.\tests\load\run_contention.ps1

Sukces: 1 aktywna rezerwacja, reszta NO_AVAILABILITY, 0× HOTEL_UPSERTED, available ≥ 0.

Mixed contention

Mixed Book + cancel + search. Pokoj jest zwalniany i rezerwowany ponownie.

.\tests\load\run_contention.ps1 -Mixed

Suma RESERVATION_CREATED moze byc > pojemnosc — oceniac aktywne rezerwacje, nie sume CREATED.

Reczny przeplyw

Get-Process locust -ErrorAction SilentlyContinue | Stop-Process -Force
python tests/load/seed_hotels.py --rooms 1
python tests/load/verify_contention.py --save-baseline
locust -f tests/load/locustfile_pure_contention.py --host http://localhost:8000 --headless -u 20 -r 20 -t 30s
python tests/load/verify_contention.py --seeded-rooms 1 --hotel h-waw-1 --room-type single

6. Weryfikacja wynikow

Skrypt verify_contention.py laczy sie jako admin i analizuje audit log oraz dostepnosc z /hotels/search.

  • Baseline — timestamp zapisany po seed, przed testem; liczone sa tylko nowe zdarzenia
  • Utworzone vs aktywne — parowanie CREATED/CANCELLED po reservation_id
  • HOTEL_UPSERTED — wykrywa restock w trakcie testu (stary Locust w tle)
  • available < 0 — double-booking w stanie magazynu

8. Typowe pulapki

  • Stary Locust w tle — AdminUser doklada pokoje (HOTEL_UPSERTED). Uzyj run_contention.ps1.
  • Backslash w PowerShell — \ nie laczy linii; uzyj jednej linii lub backtick `` ` ``.
  • Brak seed — przy malo pokoi szybko brak dostepnosci i wiszace holdy (TTL 300 s).
  • Mylenie CREATED z aktywnymi — przy mixed contention cancel zwalnia pokoje.
